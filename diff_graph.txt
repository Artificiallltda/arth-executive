    fix(graph): preserve SEND_FILE tags from inner agent tool calls in outer state
    
    create_react_agent stores ToolMessages only in its inner state. When
    agent_node returned only result["messages"][-1], tags like <SEND_FILE:...>
    from generate_image tool results were lost if the LLM didn't repeat them.
    Now agent_node scans all inner messages and appends any missing file/audio
    tags to the final message before returning to the outer graph state.
    
    Also adds Telegram sendPhoto API response logging to surface silent failures.
    
    Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
